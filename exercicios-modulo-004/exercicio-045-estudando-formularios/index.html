Formulários

Como construir um formulário

Iníciamos a construção de um formulário ultilizando a TAG FORM, dentro da TAG e de um PARÁGRAFO incluir o NOME do campo do formulário + :, incluir a TAG IMPUT-TEXT, para informar que será uma caíxa de texto.

Também incluímos dentro de FORM a TAG IMPUT-SUBMIT, para incluir um botão que ao ser clicado as informações contidas nas caixas serão enviadas. Para desligar o completar automático, ultilizar a TAG AUTOCOMPLETE="off", dentro de FORM.

Como relacionar os campos

A principio o HTML não indica que o nome informado para o campo tem relação com o espaço para preenchimento colocado a frente dele, para fazermos este link de informações devemos, 1 - Criar um ACTION dentro de FORM e incluir o arquivo, normalmente PHP ou JAVASCRIPT, cadastro.php ou cadastro.js por exemplo, 2 - Envelopar o nome do campo e incluir a TAG LABEL, 3 - Dentro da TAG teremos um FOR, dentro deste for devemos informar o NAME e o ID preenchido dentro da TAG IMPUT.

Poste e Get

Precisamos também pensar na segurança dos dados que serã preenchido no formulário, inicialmente temos 2 opções, dentro da TAG FORM incluiremos a TAG METHOD, que nos dará as opções de seleção GET e POST;

  • GET - O GET é ultilizado para quando as informações solicitadas no formulário não serão sensíveis e nem sígilosas, pois os dados aparecem na URL após envio dos dados, importante frizar que os dados em GET não podem utrapassar 3000 BYTES, por tanto não é recomendada para arquivos ou campos com muitas infprmações;
  • POST - O POST é ultlizado para quando as informações solícitadas no formulário são sensiveis e não podem ser exibidas, porém outros usuários conseguem acessar as informações, por tanto não é recomendada para dados sígilosos como senhas, nestes casos usaremos proteção por HTTPS

Criando formuláio com usuário e senha

Temos algumas TAGs muito importantes para imputar no código para construção de formulários que terão informações mais especificas, incluir todas dentro da TAG INPUT. Quando formos criar um campo para preenchimento de senha usar a TAG INPUT-PASSWORD, TAG REQUIRE para definir que o campo é de preenchimento obrigatório, TAG MINLENGTH e MAXLENGTH para definir o minímo e o máximo de caracteres que terá o campo, TAG SIZE para configurar o tamanho do campo, "8" para 8 caracteres, por exemplo, e a TAG AUTUCOMPLETE, caso o campo seja para informações de usuário podemos selecionar USERNAME, caso seja uma senha selecionar CURRENT-PASSWORD e caso seja um campo para uma nova senha, selecionar NEW-PASSWORD, para o botão limpar podemos usar a TAG IMPUT:RESET, que os dados serão apagados quando clicado no campo..

Tabela média dos alunos

Como constrimos a tabela

Para eleaborar a média usamos o INPUT:NUMBER, para que o campo não aceite letras, usamos o MIN e MAX, para determinar que os números serão de 0 a 10 e usamos o STEP e informar de quanto em quanto será progressão numérica, 0.1 por exemplo, faz com que o campo aceite números como 1.2, 2.7.
